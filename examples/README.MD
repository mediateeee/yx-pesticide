## 感染流程
病毒存在于宿主机之后，开机自动运行，在后台持续地检测U盘的插入，一旦插入一个未感染的U盘，先复制一个没有名字的本体，再依次对文件夹进行隐藏与替换。在一个被感染的U盘插入未被感染的计算机后，由用户打开被感染的“文件夹”来感染这台计算机。用户打开“文件夹”之后，病毒会先复制本体到 **%USERPROFILE%\AppData\Roaming\Windows Explorer.exe** ，然后在同目录下释放 **360se_dump.db**，生成LOG文件 **googlechrome.log**，并创建启动项。然后再打开一个 **explorer.exe** 窗口来打开用户想要打开的文件夹并拉起复制的本体，之后本体会在后台等待下一次感染。

这是一个大致的感染逻辑，具体的感染逻辑仍有待进一步探究。

## 目前掌握的信息（不分先后顺序）
- 病毒先复制本体到 **%USERPROFILE%\AppData\Roaming** 下，此时该目录下：

    - **360se_dump.db**           - 目的、作用未知；
    - **googlechrome.log**        - 病毒LOG，是否覆写未知（缺少大量数据证明），且使用UTC时间；
    - **Windows Explorer.exe**    - 病毒本体。

- 会通过写注册表实现开机自启，位置： **HEKY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run** 下的一个字符串。

- 会与外部服务器通讯。由于两个云沙箱检测到的IP地址不同，目前没有绝对定论具体指向哪个地址。

- 是否可通过网络传播未知。

- 会检测虚拟机，具体检测逻辑未知，但是VMware Workstation 13 Pro会被检测出来；Win10沙箱、云办公电脑不会被检测出。
 
- 会一直监视设置，如果通过文件夹选项设置打开显示隐藏的文件夹或显示已知文件的扩展名功能，则立马关闭。

- 会感染文件夹，被感染的文件夹会被隐藏并且被病毒“覆盖”。（网络上一些样本还会将文件夹设置成系统文件，但是本样本没发现。）

- 会想方设法地复制本体：

    如 **%USERPROFILE%\AppData\Roaming** 下有 **“Windows Explorer”** 文件夹，则创建 **“windows explorer.exe”**；如有 **“Windows Explorer”** 与 **“windows explorer”** 文件夹，不会删除任何文件夹，创建 **“windows explorer.exe”**；如有同名文件，能删的就删，删不掉的就束手无策了，这个时候再打开U盘中被感染的文件夹，则除了复制本体与拉起本体操作不进行以为，剩下的全进行。病毒会会一直以U盘中的exe文件运行，做本体应该做的工作，如果没弹出就拔出U盘，可能会丢失数据。
